# Contributing to Metton

Thank you for your interest in contributing to Metton. We welcome bug fixes, performance improvements, and documentation updates from the community.

Before diving in, please take a few minutes to read this guide.

---

## Contributor Licence Agreement

By submitting a pull request, you agree to the terms of our [Contributor Licence Agreement](CLA.md). Please read it before contributing. You retain ownership of your code, the CLA simply gives us the rights needed to distribute your contribution as part of Metton, including under commercial licences.

**If you do not agree to the CLA, please do not submit a pull request.**

---

## What We Welcome Right Now

We are in active early development. To keep the project focused, we are currently prioritising:

- **Bug fixes** — anything that causes incorrect behaviour, crashes, or data issues
- **Performance improvements** — measurable improvements to query performance, API response times, or frontend rendering
- **Documentation** — corrections, clarifications, missing setup steps, or improved code comments

We are **not** accepting new feature contributions at this stage. If you have a feature idea, please open a Discussion rather than a PR — we want to have the conversation before any code is written.
